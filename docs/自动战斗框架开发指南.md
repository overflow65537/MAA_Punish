# 自动战斗框架开发指南（agent）

本文档面向在本仓库内开发/扩展**自动战斗**的脚本作者。当前框架位于 `agent/action/combat/`，由三部分构成：

- **框架层（动作与识别）**：`agent/action/combat/actions/CombatActions.py`
- **配置层（角色注册）**：`agent/action/combat/config/LoadSetting.py`（`ROLE_ACTIONS`）
- **业务层（角色状态机）**：`agent/action/combat/roles/*.py`（继承 `BaseRole`）

战斗循环由 **`CombatRunner`**（`agent/action/combat/entry/runner.py`）统一驱动，Pipeline 只需调用一次 `custom_action: "CombatRunner"`。新角色**无需**在 `agent_file.py` 里逐个注册，只要 `cls_name` 与类名一致，`role_factory` 会自动扫描 `roles/` 包。

---

## 1. 运行机制概览

```
Pipeline（CombatRunner）
  → CombatTask.combat_once()
    → 进战识别队伍 / attack_template 校正 cls
    → 循环：当前角色 role.perform() → do_perform()
      → CombatActions（攻击/消球/识别/QTE）
      → request_role_switch() 切人
```

每个 tick，`BaseRole.perform()` 会先处理切人阶段与 idle 校验，再进入子类实现的 `do_perform()`。业务逻辑本质是：**按优先级判断状态 → 调用动作节点 → 必要时切人**。

| 路径 | 说明 |

## 3. 新增角色：LoadSetting

在 `ROLE_ACTIONS` 中新增一项，**`cls_name` 必须与 `roles/` 下类名完全一致**。

### 3.1 常用字段

- **键名**（如 `"神威·不落日"`）：正式名称，选人/OCR 用
- **`name`**：简称别名
- **`type`**：`Attacker` / `Tank` / `Support`（破甲算 `Tank`，增幅算 `Support`）
- **`cls_name`**：如 `"Aeternion"`
- **`metadata`**：元素与代数，如 `{"fire": 100, "generation": 3}`
- **`template`**：编队头像模板列表
- **`attack_template`**：战斗内攻击键模板（可多个，用于 p1/p2 识别）
- **`skill_template`**（可选）：三色球模板，供 `Arrange_Signal_Balls()` 三消决策

### 3.2 示例

```python
"神威·不落日": {
    "name": "不落日",
    "type": "Tank",
    "cls_name": "Aeternion",
    "metadata": {"fire": 100, "generation": 3},
    "template": [
        "人物索引/神威/不落日/不落日.png",
        "人物索引/神威/不落日/不落日_终解.png",
    ],
    "attack_template": [
        "自定义战斗/神威/不落日/不落日.png",
        "自定义战斗/神威/不落日/不落日_释能.png",
    ],
},
```

## 4. 新增角色：roles 业务脚本

### 4.1 约定

- 文件：`agent/action/combat/roles/aeternion.py`
- 类名：`class Aeternion(BaseRole)`（与 `cls_name` 一致）
- 入口：实现 `do_perform()`，用 `self.phase` 做状态机
- 切人：`self.combat.request_role_switch(self)`（由 `CombatTask` 统一处理 QTE）
- 动作：`self.action.*`（`CombatActions` 实例）

### 4.2 最小模板

```python
from action.combat.core.role import BaseRole


class YourRole(BaseRole):
    def do_perform(self) -> None:
        if self.combat.context.tasker.stopping:
            return

        if self.phase == "idle":
            self.action.lens_lock()
            self.phase = "combat"
            return

        if self.action.check_Skill_energy_bar():
            self.action.use_skill_until_empty()
            self.action.use_qte()
            self.combat.request_role_switch(self)
            return

        self.action.continuous_attack(8, 50)

    def on_switch_failed(self) -> None:
        self.phase = "combat"
```

### 4.3 编写建议

- **状态机写在模块 docstring**，便于 LLM/他人阅读（参考 `arete.py`、`aeternion.py`）
- **防发呆**：关键步骤用 `_run_step`（首尾 `attack()`）包裹；消球/长按前后补普攻
- **识别优先**：条件判断走 `check_status("节点名")`，节点定义在 `Check_Characters_Skill/<Cls>.jsonc`
- **双形态**：攻击键 `TemplateMatch` ROI 通常与 `[1113, 556, 138, 142]` 一致；Win32 可复用同 ROI
- **切人**：大招后常见顺序为 `use_skill_until_empty()` → `use_qte()` → `request_role_switch()`

## 5. Pipeline 与识别节点

### 5.1 动作节点

定义于 `resource/base/pipeline/Auto_Battle/Auto_Battle_Action.jsonc`（ADB）与 `resource/pgr_win32/pipeline/Auto_Battle_action.jsonc`（Win32）：

`攻击`、`闪避`、`技能`、`长按攻击`、`长按闪避`、`消球1`~`消球8`、`qte1`、`qte2`、`锁定视角`、`辅助机` 等。

### 5.2 角色特判

每个角色可在 `Check_Characters_Skill/<ClsName>.jsonc` 增加节点，例如：

- `检查阶段p1_不落日` / `检查阶段p2_不落日`：`TemplateMatch` + 攻击键模板
- `检查p1核心条_不落日`：`ColorMatch` + ROI/HSV

业务中调用：

```python
if self.action.check_status("检查p1核心条_不落日"):
    ...
```

### 5.3 Win32 覆写

`resource/pgr_win32/pipeline/Auto_Battle/Check_Characters_Skill/` 下同名 jsonc **只需写 ROI**，色域/模板继承 base。攻击键阶段 ROI 通常与 base 相同；核心条等 UI 需 Win32 实机标定。

## 6. CombatActions 常用 API

| 方法 | 说明 |
|------|------|
| `attack()` / `continuous_attack(n, ms)` | 普攻 |
| `dodge()` / `long_press_dodge(ms)` | 闪避 |
| `long_press_attack(ms)` | 长按普攻 |
| `use_skill()` / `use_skill_until_empty()` | 大招 |
| `ball_elimination_target(slot)` | 消指定位球（1~8） |
| `count_signal_balls()` | OCR 球数 |
| `Arrange_Signal_Balls("red"\|"blue"\|"yellow"\|"any")` | 三消决策（需 `skill_template`） |
| `check_Skill_energy_bar()` | 大招能量条 |
| `check_status(node)` | 跑 Pipeline 识别节点 |
| `use_qte()` | 同时 qte1 + qte2 |
| `auxiliary_machine()` | 辅助机 |
| `lens_lock()` | 锁定视角 |

切人请用 **`self.combat.request_role_switch(self)`**，不要直接拼 QTE 坐标。

## 8. 调试与常见问题

- **角色不生效**：检查 `cls_name` 是否与类名一致、是否继承 `BaseRole`、`LoadSetting` 是否登记
- **识别不命中**：用 `check_status(node, pipeline_override={...})` 临时调 ROI；稳定后写入 jsonc
- **Win32/ADB 差异**：模板需分平台截图；Pipeline ROI 在 `pgr_win32` 覆写
- **三消无效**：确认 `skill_template` 已配置且 `role_name` 能匹配到 `ROLE_ACTIONS`
